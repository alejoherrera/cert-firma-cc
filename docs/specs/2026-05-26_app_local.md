# 2026-05-26 — App local (Streamlit) para uso unipersonal del jefe

## Cumplimiento constitucional
Cumple §1 (propósito) y §6 (rol "operador del pipeline = jefe del Centro"). Habilita ejecución sin línea de comandos del pipeline existente. Scope expansion de la spec madre `2026-05-26_acta_hash_certificados.md`.

## Problema
El pipeline actual requiere abrir terminal y ejecutar 4 scripts en orden. Eso fricciona al jefe del Centro cuando solo quiere "procesar la nueva cohorte". Para que cuando llegue un sucesor pueda usarlo sin entrenamiento técnico, conviene una UI con wizard guiado.

## Solución propuesta

Aplicación local en **Streamlit**, archivo único `app/app.py`. Arranque por `iniciar.bat` (doble-click). Asume Python instalado en la máquina (v0.3.0). Empaquetado standalone con PyInstaller queda para v0.4.0.

**UX wizard de 4 pasos:**

1. **Estado de conexión Drive**: muestra si hay token CGR válido (`C:\Users\aleja\token_cgr.json`). Si expiró, instrucciones para refrescar. Sin OAuth flow interactivo en v0.3.0.
2. **Elegir cohorte**: lista las subcarpetas dentro de "Firma" del Drive. Usuario elige una.
3. **Preview**: lista los N PDFs encontrados + extracción de 3 al azar para verificación visual.
4. **Procesar**: input para nombre del curso. Botón "Procesar lote". Barra de progreso. Al terminar, muestra:
   - Path local del lote
   - Botón "Abrir carpeta en Windows Explorer"
   - Botón "Abrir acta.pdf"
   - Botón "Descargar lote.zip"

No modifica scripts/*: la app los importa.

## Criterios de aceptación

1. `iniciar.bat` (doble-click) levanta Streamlit en `localhost:8501` y abre navegador.
2. App detecta token CGR existente y muestra "Conectado".
3. Lista subcarpetas de la carpeta raíz Drive (`1U7aSbdtoPl3cYVWDzff5_LQwQ8Bjk_1T`).
4. Procesa lote completo: genera mismos outputs que `python scripts/generar_acta.py` + `generar_pdf_acta.py` + `generar_excel_acta.py`.
5. **Smoke**: la app arranca sin errores; click en "procesar" sobre cohorte ya bajada produce un nuevo `lote_<timestamp>/` con los 19 archivos esperados (15 PDFs + acta.pdf + acta.xlsx + listado.csv + .json + manifest.json + run.log).

## Out of scope

- OAuth flow interactivo (token se obtiene out-of-band).
- Subida de outputs a Drive.
- Empaquetado `.exe` con PyInstaller (queda para v0.4.0).
- Multi-usuario / concurrencia (es unipersonal — el jefe del Centro).
- Firma BCCR automatizada (el jefe firma `acta.pdf` aparte, como ya).
